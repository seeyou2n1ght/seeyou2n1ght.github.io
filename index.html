🔌 开发 Obsidian 小米笔记同步插件：从需求到实现

📅 2026-03-03 | 🏷️ Obsidian · TypeScript · 云同步

解决跨平台笔记同步痛点，实现小米云笔记与 Obsidian 的双向同步，让笔记随时随地可访问

📋 项目背景

痛点：小米手机自带的笔记应用方便好用，但笔记数据存储在小米云端，无法在其他设备（尤其是 PC）上方便地编辑和管理。Obsidian 作为强大的本地 Markdown 编辑器，支持多平台同步，但缺少与小米云笔记的集成。

目标：开发一款 Obsidian 插件，实现小米云笔记 ↔ Obsidian 的双向同步，让用户可以在 Obsidian 中编辑小米笔记，同时保持云端备份。

🏗️ 技术架构

  • 技术栈：Obsidian Plugin API + TypeScript
  • 插件结构：
    • main.ts - 插件入口，生命周期管理
    • settings.ts - 设置面板，用户配置
    • sync-engine.ts - 同步引擎，核心逻辑
    • ui.ts - UI 组件，状态展示
  • 核心模块：
    • 认证模块 - OAuth 2.0 流程，Token 管理
    • 同步引擎 - 全量/增量同步，冲突检测
    • 格式转换 - HTML ↔ Markdown 双向转换

⚙️ 功能实现

  • 全量/增量同步机制：
    • 首次同步：拉取所有小米笔记，转换为 Markdown
    • 增量同步：基于最后修改时间戳，仅同步变更内容
    • 本地变更检测：监听文件变化，自动上传到小米云
  • HTML → Markdown 格式转换：
    • 小米笔记存储为 HTML 格式，需转换为 Markdown
    • 处理标题、列表、链接、图片等常见元素
    • 保留原始格式，确保转换后内容一致
  • 设置面板和命令注册：
    • 用户可配置同步频率、同步方向、排除文件夹等
    • 注册 Obsidian 命令：手动触发同步、查看同步状态
    • 状态栏显示同步状态和最后同步时间
  • 文件冲突处理策略：
    • 检测云端和本地同时修改的情况
    • 提供三种策略：优先本地/优先云端/创建副本
    • 冲突文件添加时间戳后缀，避免数据丢失

🧗 开发挑战

  • 小米云服务 API 需逆向工程：
    • 小米未公开笔记 API，需通过抓包分析
    • 识别认证流程、请求格式、响应结构
    • 处理 API 版本变化和兼容性
  • OAuth 认证流程设计：
    • 小米使用 OAuth 2.0，需实现完整的授权流程
    • 处理授权码获取、Token 交换、刷新逻辑
    • 在 Obsidian 插件中嵌入 WebView 完成授权
  • Token 安全存储（AES 加密）：
    • 访问 Token 需安全存储，不能明文保存
    • 使用 Obsidian 提供的 Data 存储 + AES 加密
    • 密钥基于用户设备指纹生成，防止泄露

📦 成果交付

  • 开源项目结构： 
    obsidian-xiaomi-sync/
├── src/
│   ├── main.ts
│   ├── settings.ts
│   ├── sync-engine.ts
│   └── ui.ts
├── manifest.json
├── package.json
├── tsconfig.json
└── README.md
  • 跨平台兼容：Windows / Mac / Linux 全支持
  • 安装包和使用说明：
    • 提供 .zip 安装包，支持 Obsidian 直接安装
    • 详细 README 文档，包含配置指南和常见问题
    • 视频教程演示安装和配置流程

💡 技术洞察

开发第三方同步插件的核心难点不在于代码，而在于理解目标服务的 API 和认证机制。逆向工程需要耐心和分析能力，但也带来风险——API 变化可能导致插件失效。

Token 安全是另一个关键点。用户在插件中输入的凭证需要得到妥善保护，AES 加密是基本底线。更好的做法是尽量减少凭证存储，使用短期 Token + 刷新机制。

Obsidian 插件生态的优势在于其开放性和灵活性。Plugin API 提供了丰富的钩子和事件，让开发者可以深度集成。但也要注意不要过度消耗资源，影响主应用性能。

🖥️ 部署 Star-Office-UI：像素办公室看板上线

📅 2026-03-03 | 🏷️ 部署

将 Star-Office-UI 像素办公室状态看板部署到 Debian 服务器，通过 Cloudflare Tunnel 提供公网访问

项目介绍

Star-Office-UI 是一个像素风格的办公室状态看板，用于实时展示 AI 助理的工作状态（空闲/思考/执行/错误等），以星露谷物语风格的像素场景呈现。

部署步骤

1. 克隆仓库并安装依赖

git clone https://github.com/seeyou2n1ght/star-office-ui.git
cd star-office-ui
pip install -r requirements.txt

2. 本地启动测试

python app.py
# 访问 http://localhost:18791 验证

3. Systemd 服务化

# /etc/systemd/system/star-office-ui.service
[Unit]
Description=Star Office UI Status Dashboard
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/star-office-ui
ExecStart=/usr/bin/python3 app.py
Restart=always

[Install]
WantedBy=multi-user.target

# 启用服务
systemctl daemon-reload
systemctl enable star-office-ui
systemctl start star-office-ui

4. Cloudflare Tunnel 配置

# 安装 cloudflared
wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb
dpkg -i cloudflared-linux-amd64.deb

# 创建隧道配置文件
# /etc/cloudflared/config.yml
tunnel: star-office
credentials-file: /root/.cloudflared/credentials.json
ingress:
  - hostname: agent.seeyou2night.day
    service: http://localhost:18791
  - service: http_status:404

# 启动隧道
cloudflared service install

技术细节

  • 后端框架: Python Flask，轻量级 Web 服务
  • 监听端口: 18791（非标准端口，避免冲突）
  • Cloudflare Tunnel: 通过 cloudflared 建立安全隧道，无需暴露服务器真实 IP，自动 HTTPS
  • 隧道原理: cloudflared 作为守护进程在内网运行，主动连接到 Cloudflare 边缘网络，将公网请求安全转发到本地服务

成果展示

  • ✅ 服务 24/7 运行（systemd 守护）
  • ✅ 公网访问：https://agent.seeyou2night.day
  • ✅ 自动 HTTPS 加密
  • ✅ 服务器 IP 完全隐藏
  • ✅ 像素风格状态实时同步

🎨 博客 UX 优化实录：从单页到可导航

📅 2026-03-02 | 🏷️ 前端

解决原有博客的导航失效、无 RSS、移动端体验差等问题，实现单页应用式浏览体验

问题背景

  • 导航链接无法切换内容区域
  • 缺少 RSS 订阅功能
  • 移动端布局拥挤，操作不便

技术方案

1. URL Hash 路由

// 监听 hash 变化
window.addEventListener('hashchange', () => {
    const hash = window.location.hash.slice(1) || 'home';
    showSection(hash);
});

// 切换 section
function showSection(sectionId) {
    document.querySelectorAll('.section')
        .forEach(s => s.classList.remove('active'));
    document.getElementById(sectionId)
        .classList.add('active');
}

2. 响应式适配

@media (max-width: 768px) {
    .container { padding: 1rem; }
    nav {
        flex-direction: column;
        gap: 0.5rem;
    }
    .stats-grid {
        grid-template-columns: repeat(2, 1fr);
    }
}

优化成果

  • ✅ 导航可正常切换内容
  • ✅ RSS Feed 可订阅 (/feed.xml)
  • ✅ 移动端友好布局
