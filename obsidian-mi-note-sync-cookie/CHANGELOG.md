# 更新日志

## [2.0.0] - 2026-03-03

### 🎉 重大更新 - Cookie 认证版本

#### ✨ 新增功能

**认证系统重构**
- ✅ 完全移除 OAuth 认证流程
- ✅ 新增 Cookie 认证模块 (`auth/cookie-auth.ts`)
- ✅ 实现 AES-256 加密存储
- ✅ 自动检测 Cookie 过期
- ✅ 401/403 错误自动处理

**用户界面**
- ✅ 新增登录窗口 UI (`ui/login-modal.ts`)
- ✅ 设置面板更新（显示登录状态和 Cookie 有效期）
- ✅ 登录指引说明
- ✅ 安全提示

**自动化**
- ✅ 同步前自动检查 Cookie 有效性
- ✅ 过期自动提示重新登录
- ✅ 自动同步时认证检查

#### 🔧 技术改进

**代码结构**
- 重构认证模块，分离 Cookie 管理逻辑
- 优化 API 请求处理，统一认证头
- 改进错误处理机制

**安全性**
- Cookie 使用 AES-256 加密
- 基于设备标识生成加密密钥
- 不存储用户名和密码
- 仅保存必要的 serviceToken

**性能**
- 减少不必要的 Token 刷新请求
- 优化认证检查流程
- 提高同步效率

#### 📝 文档更新

- ✅ 重写 README.md
- ✅ 新增 DEVELOPMENT.md（开发文档）
- ✅ 新增 IMPLEMENTATION_REPORT.md（实现报告）
- ✅ 新增 QUICK_START.md（快速上手）
- ✅ 新增 CHANGELOG.md（更新日志）

#### 🐛 Bug 修复

- 修复 OAuth Token 刷新失败问题
- 修复认证状态不同步问题
- 修复设置面板刷新问题

#### ⚠️ 破坏性变更

**从 v1.x 升级需要注意：**

1. **认证方式变更**: OAuth Token → Cookie
   - 需要重新登录
   - 旧 Token 数据将不再使用

2. **设置结构变更**:
   ```typescript
   // 旧版
   {
     accessToken: string;
     refreshToken: string;
     tokenExpiry: number;
   }
   
   // 新版
   {
     encryptedCookies: string;
     cookieExpiry: number;
     lastCookieRefresh: number;
   }
   ```

3. **升级步骤**:
   - 备份现有设置
   - 安装新版本插件
   - 重新登录小米账号
   - 配置同步设置

## 版本说明

### 版本号规则

遵循语义化版本（Semantic Versioning）：`主版本。次版本.修订版本`

- **主版本**: 破坏性变更
- **次版本**: 向后兼容的功能新增
- **修订版本**: 向后兼容的问题修复

### 支持周期

- **v2.x**: 当前支持版本
- **v1.x**: 已停止支持（2026-03-03 起）

### 升级建议

**强烈建议所有用户升级到 v2.0.0**

理由：
- 更简单的登录流程
- 更好的安全性
- 更稳定的认证机制
- 持续的技术支持
