# 快速使用指南

## 🚀 5 分钟快速上手

### 1️⃣ 安装插件

```bash
# 方法一：直接复制构建产物
cp -r /root/.openclaw/workspace/obsidian-mi-note-sync-cookie \
  /path/to/your/obsidian/vault/.obsidian/plugins/mi-note-sync-cookie

# 方法二：使用构建产物
cd /root/.openclaw/workspace/obsidian-mi-note-sync-cookie
npm run build
# 将 main.js, manifest.json, styles.css 复制到插件目录
```

### 2️⃣ 启用插件

1. 打开 Obsidian
2. 进入 **设置** → **第三方插件**
3. 找到 **Mi Note Sync (Cookie)**
4. 点击 **启用**

### 3️⃣ 登录小米账号

1. 进入插件设置面板
2. 点击 **🔐 登录** 按钮
3. 在打开的浏览器中访问 https://i.mi.com/
4. 使用小米账号登录
5. 返回 Obsidian，点击 **✅ 我已登录**

### 4️⃣ 配置同步

1. **同步文件夹**: 设置笔记保存位置（默认：`小米笔记`）
2. **同步模式**: 选择全量或增量同步
3. **冲突处理**: 选择覆盖、跳过或重命名
4. **自动同步**: 可选启用（默认关闭）

### 5️⃣ 开始同步

**方法一**: 点击 Ribbon 图标（同步按钮）

**方法二**: 使用命令面板
```
Ctrl/Cmd + P → 小米笔记：同步所有小米笔记
```

**方法三**: 使用命令
- `小米笔记：从小米云拉取笔记`
- `小米笔记：推送笔记到小米云`

## 🔧 设置说明

### 账号设置

- **登录状态**: 显示当前登录的小米账号
- **Cookie 有效期**: 显示 Cookie 剩余有效期
- **登录/退出**: 管理登录状态

### 同步设置

- **同步文件夹**: Obsidian 中保存笔记的文件夹路径
- **同步模式**:
  - `全量同步`: 每次同步所有笔记
  - `增量同步`: 仅同步变更的笔记（推荐）
- **冲突处理**:
  - `覆盖`: 用新笔记覆盖现有笔记
  - `跳过`: 保留现有笔记
  - `重命名`: 自动重命名新笔记（推荐）

### 自动同步

- **启用自动同步**: 定期自动同步
- **同步间隔**: 设置同步频率（分钟）

### 标签过滤

- **包含标签**: 只同步包含指定标签的笔记
- **排除标签**: 不同步包含指定标签的笔记

## 💡 使用技巧

### 1. 首次同步建议

- 使用 **全量同步** 模式
- 确保网络连接稳定
- 等待同步完成后再进行操作

### 2. 日常使用

- 启用 **增量同步** 提高效率
- 设置 **自动同步**（建议 30 分钟）
- 使用 **插入笔记** 命令快速引用

### 3. 冲突处理

- 推荐设置为 **重命名**
- 避免数据丢失
- 手动检查冲突笔记

### 4. 标签管理

- 使用标签过滤提高同步效率
- 排除临时笔记和草稿
- 只同步重要笔记

## ⚠️ 注意事项

### 登录相关

- Cookie 有效期为 30 天
- 过期后需要重新登录
- 不要在多个设备同时登录同一账号

### 同步相关

- 首次同步可能需要较长时间
- 大附件同步较慢
- 建议在 WiFi 环境下同步

### 安全相关

- Cookie 已加密存储
- 不要分享插件数据文件
- 定期备份重要笔记

## 🐛 常见问题

### Q: 登录失败怎么办？

**A**: 
1. 检查网络连接
2. 确认小米账号密码正确
3. 尝试在浏览器中登录 https://i.mi.com/
4. 重新打开登录窗口

### Q: Cookie 过期了怎么办？

**A**: 
1. 进入插件设置
2. 点击 **退出登录**
3. 重新 **登录**

### Q: 同步失败怎么办？

**A**: 
1. 检查登录状态
2. 检查网络连接
3. 查看开发者控制台错误信息
4. 尝试重新同步

### Q: 笔记内容乱码怎么办？

**A**: 
1. 检查笔记编码格式
2. 尝试重新拉取笔记
3. 联系开发者反馈

### Q: 可以双向同步吗？

**A**: 
目前主要支持从小米云拉取笔记，推送功能正在开发中。
